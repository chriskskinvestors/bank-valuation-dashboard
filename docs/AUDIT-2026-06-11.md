## B. SILENT-FAILURE POSTURE (Goal-Driven Execution)

- **214 `except Exception` clauses** repo-wide; **41 bare `pass`**; ~96 in data/ alone of which **~50 are accidental** "API failure → quietly empty" (vs ~15 deliberate, documented fail-opens which are fine and well-marked).
- The ONLY error channel from data layer to UI is *absence of data*; observability is `print()` to Cloud Run logs.
- Per-cell `except Exception → "—"` in both statement engines makes a typo'd FDIC field permanently indistinguishable from "not reported" (financial_highlights.py:600-603; financials_statements.py:322-325) — zero logging.
- 16 dead `except ImportError` plotly shims (plotly is a hard dependency; other modules import it unconditionally) — unreachable fallbacks, some of which would render *nothing* silently if reachable.
- The critical fetch with NO retry: sec_client._download_company_facts (one attempt, `except → {}`) — every SEC fundamental depends on it, while 4 different retry implementations exist elsewhere.
